# دليل تطوير نظام الـ Micro-Frontends (HIS/EMR)

هذا المستند يلخص ما تم إنجازه في هيكلية النظام وطريقة العمل (Workflow) لإضافة وتطوير الموديولات.

## 1. الهيكل العام للنظام (Architecture Overview)
النظام يعتمد على تقنية **Native Federation** مع **Angular 18**، وهي التقنية الأحدث التي تسمح بتقسيم التطبيق إلى أجزاء مستقلة تعمل معاً ككتلة واحدة في المتصفح.

### المكونات الرئيسية:
1.  **Host Shell (Port 8080)**: التطبيق الرئيسي الذي يحتوي على القوائم، التبويبات، والداشبورد.
2.  **EMR Shell (Port 8090)**: موديول متخصص لفتح ملفات المرضى، ويدعم نظام التبويبات المتعددة (Multi-Patient Tabs).
3.  **Central MFE (Port 8083)**: موديول يحتوي على قوائم المرضى (Inpatients/Outpatients).
4.  **Patient MFEs (10+ Modules)**: موديولات طبية دقيقة (Vitals, Notes, Labs, etc.) يتم تحميلها عند الحاجة.

## 2. ما تم إنجازه (Achievements)
✅ **بناء الهيكل الأساسي**: إعداد 14 موديولاً مستقلاً تماماً.
✅ **نظام التبويبات الديناميكي**: القدرة على فتح أكثر من مريض في وقت واحد دون تداخل البيانات.
✅ **التواصل العابر للموديولات**: تنفيذ `Event Bus` يسمح للموديولات بتبادل البيانات لحظياً (مثل تحديث الملاحظات عند تغيير العلامات الحيوية).
✅ **التكامل مع Docker**: إعداد بيئة بناء وتشغيل موحدة تضمن عمل النظام في أي مكان بنفس الإعدادات.
✅ **إصلاحات تقنية متقدمة**: حل مشاكل الـ BOM، وتعديل إعدادات Nginx لدعم الـ Routing والـ CORS.

## 3. طريقة العمل (Development Workflow)

### كيف تضيف موديول جديد؟
1.  **إنشاء المشروع**: إنشاء مشروع Angular جديد داخل المجلد الرئيسي.
2.  **إعداد الـ Federation**: إضافة مكتبة `@angular-architects/native-federation` وضبط ملف `federation.config.js`.
3.  **التسجيل في الـ Shell**: إضافة مسار الموديول الجديد في ملف `public/federation.manifest.json` داخل الـ Shell.
4.  **الـ Dockerization**: نسخ ملف `Dockerfile` و `nginx.conf` من أي موديول موجود وتعديل اسم المشروع.
5.  **التحديث في Docker Compose**: إضافة الخدمة الجديدة في ملف `docker-compose.yml` وتحديد البورت.

## 4. تقنيات التواصل المستخدمة
| النوع | التقنية | الاستخدام |
| :--- | :--- | :--- |
| **Navigation** | `postMessage` | لفتح تابة جديدة أو الانتقال لصفحة في موديول آخر. |
| **Sync** | `CustomEvents` | لتحديث البيانات بين موديولين مفتوحين في نفس الشاشة. |
| **Context** | `loadRemoteModule` | لتحميل المكونات ديناميكياً داخل الـ Shell. |

## 5. تشغيل النظام
للبدء، استخدم الأوامر التالية من المجلد الرئيسي:
```powershell
# لبناء وتشغيل كافة الموديولات
docker-compose up -d --build

# لإيقاف النظام
docker-compose down
```
